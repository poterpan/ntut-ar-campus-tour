## 關鍵事實庫(寫報告用)
- 場域:新生南路側門,5 POI(側門/學餐/演講廳/第一教學大樓/化工館)
- 定位精度:校外水平 <0.5m、校內 <1m、方位角誤差 <1°(提案階段實測)
- NPC:Jenson(Meshy biped),動畫 聆聽/說話/走動/揮手
- LLM:OpenAI 相容 proxy(testvideo,gpt-5.5);語音 GLM TTS 小陳 + ElevenLabs scribe STT + OpenCC 簡繁
- RL(**以實際 shipped code 為準**):**tabular Q-Learning**,狀態 = 距離 4 區間 × 角度 8 區間 = **32 離散狀態**,動作 3(直走/左轉/右轉),reward = 距離 shaping(Δdist×2)+ 時間 -0.05 + 對齊 ±0.02 + 吃到 +15 +(訓練)出界 -10;α=0.1 γ=0.9 ε 1.0→0.05 decay 0.995;**1300 episodes** Q-table(commit ad58…/`Assets/ML/cat_qtable.json`,蔡訓練)。📌 對老師的 RL 敘事:ML-Agents 是課程作業(訓練 Unity 蜂鳥 Hummingbirds)用過的工具;本專案**評估後認為此低維單一目標任務不需那麼重的神經網路模型**,故**嘗試另一種更輕量的經典 RL 方法——自寫 tabular Q-Learning**。⚠️ **不要講 protobuf 衝突(那是錯的)**。
- 平台:iOS(TestFlight)+ Android(GitHub pre-release),CI 自動觸發,金鑰 secret 注入
- 主要踩雷(第 6 章,報告留 5 個):AR 相機黑屏雙重成因(相機權限 + 隱私窗漏掉 SwitchToARView 啟用 AR session)、Android Vulkan→GLES3 黑屏、LLM 穩定性改善(429/逾時 → 重試+容錯)、簡繁歧義(OpenCC s2tw)、GLM 提示音能量裁剪。⚠️ 已移除(使用者要求):protobuf 衝突、LLM 403(供應商問題)、app icon 分裂、iOS 簽名
- **即時性痛點(討論 + 未來展望都要提):** 對話走 STT→LLM→TTS 三段雲端 API 串接,每段各自的網路+推論時間累積 → 端到端反應偏慢,尤其 **LLM 生成完整回答後才送 TTS** 這段最明顯(Demo 看得出停頓)。緩解方向:串流式 LLM/TTS(邊生成邊念、首句先回)、減少串接段數、改用端側或單一供應商。老師可能會問,務必主動說明。
