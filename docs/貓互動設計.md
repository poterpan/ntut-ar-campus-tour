## RL 規格(以「最小可運作」為原則)

### 觀察空間 — `[1, 7]` float

關鍵原則:

1. **用貓本地座標**(以貓為原點、貓朝前為 +Z),不要用世界座標 —— policy 對位移 / 旋轉不變,訓練快很多
2. 所有數值**正規化到 ~[-1, 1] 或 [0, 1]**
3. 只看 X-Z 平面(AR 平面),Y 不入 obs

| # | 欄位 | 計算方式 | 正規化 |
|---|---|---|---|
| 0–1 | `food_local_x`, `food_local_z` | 最近飼料相對貓的 local 座標 | / MAX_RANGE(預設 5m) |
| 2 | `food_dist` | sqrt(x²+z²),冗餘但幫助學習 | / MAX_RANGE |
| 3 | `food_exists` | 1.0 有 / 0.0 沒飼料 | flag |
| 4–5 | `vel_local_x`, `vel_local_z` | 自身速度(local frame) | / MAX_SPEED(預設 0.5 m/s) |
| 6 | `hunger` | 飢餓值,隨時間遞增、吃到歸零 | 已是 [0, 1] |

**進階(視訓練時間決定,起手不加):**

- 多顆飼料 → 改最近 K=3 顆 × 4 欄位 = `[1, 13]`
- 加障礙物 → 4 個方向 raycast 距離 = `[1, 11]`(原 7 + 4)

### 動作空間 — `[1, 2]` float 連續(推薦)

| # | 欄位 | 值域 | Unity 端換算 |
|---|---|---|---|
| 0 | `forward_speed` | [-1, 1] | `vel = action[0] × MAX_SPEED` |
| 1 | `turn_rate` | [-1, 1] | `rot += action[1] × MAX_TURN × dt`(MAX_TURN = 90°/s) |

選 forward + turn 而不是 `vel_x, vel_z` 的原因:貓的動作更像「走+轉頭」,在 AR 看起來自然,不會出現詭異側滑。

**離散替代(訓練時間不夠時改用):**

- shape `[1, 1]` discrete,branch size = 4
- 0 = idle、1 = forward、2 = turn-left-while-forward、3 = turn-right-while-forward
- 訓練快、policy 簡單、ONNX 小;動作不夠平滑可在 Unity 端用 Lerp 補

⚠️ 連續 → 離散的切換要 D 通知 A,因為 A 的推論端 action 解碼會不同。

### Reward 設計(初版)

| 事件 | Reward |
|---|---|
| 吃到飼料 | **+1.0**(主要 sparse 訊號) |
| 每 step | **-0.001**(輕微鼓勵效率,不要太強) |
| 撞牆 / 撞障礙 | **-0.1** |
| 接近飼料(distance 縮短) | 可選 dense reward:`+0.01 × Δdistance` |
